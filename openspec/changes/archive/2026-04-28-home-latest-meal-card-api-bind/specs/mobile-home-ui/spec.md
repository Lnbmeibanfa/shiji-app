## MODIFIED Requirements

### Requirement: 首页 MUST 以可滚动纵向布局展示五大区块

已登录用户在选中「首页」Tab 时，主内容区 MUST 提供垂直滚动容器，并按自上而下顺序包含以下逻辑区块：（1）日期与标语问候区；（2）拍照主入口；（3）今日热量与进度卡片；（4）最近记录区块；（5）AI 建议区块。各区块之间 MUST 使用项目中间距 token 保持可读留白。

#### Scenario: 用户进入首页可见完整结构

- **WHEN** 用户已登录且选中「首页」Tab
- **THEN** 主内容区可纵向滚动，并依次展示上述五个区块；最近记录区块可在渲染后异步请求数据，但 MUST NOT 阻塞首页整体展示

### Requirement: 最近记录区块 MUST 含「查看全部」并展示最近一餐卡片

最近记录区块 MUST 具备区块标题「最近记录」与右侧「查看全部」操作。标题与操作 MUST 使用 `SectionTitle`（或等价组件）实现，且「查看全部」MUST 可点击。区块内 MUST 使用标准 `MealRecordCard`（或等价业务组件）展示**一条最近一餐数据卡片**；当最近一餐接口返回 `data=null` 时，区块 MUST 展示引导去记录一餐的空态内容。点击「查看全部」MUST 将主壳当前 Tab 切换为「记录」Tab，MUST NOT 导航至记录 Tab 下的子路径或子路由栈。

#### Scenario: 展示单条最近一餐

- **WHEN** 最近一餐接口返回 `data` 非空
- **THEN** 最近记录区块可见标题、「查看全部」以及一条真实最近一餐卡片

#### Scenario: 无记录展示引导空态

- **WHEN** 最近一餐接口返回 `data=null`
- **THEN** 最近记录区块展示引导用户去记录一餐的空态文案和入口

#### Scenario: 查看全部切换到记录 Tab

- **WHEN** 用户点击「查看全部」
- **THEN** 底部导航选中「记录」Tab，主内容区切换为记录 Tab 对应页面（可为占位页）
