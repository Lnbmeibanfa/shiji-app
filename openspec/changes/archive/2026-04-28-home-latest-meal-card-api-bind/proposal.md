## Why

首页“最近记录”当前仍使用 mock 数据，用户无法看到自己真实的最近一餐热量，信息价值较低。现在后端已具备最近一餐查询 API，需要将首页卡片接入真实数据并明确空态、跳转与展示口径。

## What Changes

- 将首页“最近记录”区块从静态 mock 调整为进入页面即请求最近一餐接口。
- 最近一餐存在时，卡片展示餐别与时间（同时展示）以及热量，热量显示为整数 `kcal`。
- 最近一餐不存在时，卡片展示引导文案并提供跳转至“记录饮食”页面的交互入口。
- 保留异常降级能力：接口失败时前端仍可展示稳定占位，不阻断首页渲染。

## Capabilities

### New Capabilities

- `home-latest-meal-card`: 定义首页最近一餐卡片与后端最近一餐接口的绑定行为、展示规则与空态引导。

### Modified Capabilities

- `mobile-home-ui`: 将“最近记录区块仅 mock 渲染”的要求调整为“最近一餐卡片可使用真实接口数据并定义空态引导”。
- `meal-app-shell`: 放宽首页“不得依赖网络”的限制，允许最近一餐区块触发只读接口请求，同时要求失败可降级展示。

## Impact

- **客户端**：`apps/mobile` 首页页面、数据模型、repository/provider 需新增最近一餐查询与状态渲染逻辑。
- **接口依赖**：依赖 `GET /api/meal-records/latest` 返回 `data` 或 `null`。
- **体验**：首页信息由示例数据升级为用户真实数据；无记录用户会获得“去记录一餐”的明确入口。
