## ADDED Requirements

### Requirement: 视觉识别接口必须使用 qwen3.6-flash
系统 MUST 提供视觉理解接口处理菜品图片识别，并默认使用 `qwen3.6-flash` 作为模型标识。接口实现 MUST 通过配置读取模型名，且在未显式覆盖时使用 `qwen3.6-flash`。

#### Scenario: 默认模型生效
- **WHEN** 运行环境未设置模型名覆盖配置
- **THEN** 服务端 MUST 使用 `qwen3.6-flash` 发起模型调用

### Requirement: 视觉识别结果必须结构化返回
系统 MUST 将模型输出转换为结构化响应，至少包含菜品名称候选、食材列表候选与处理状态字段。服务端 MUST NOT 直接透传不可解析的自由文本给客户端作为主结果。

#### Scenario: 识别成功返回结构化数据
- **WHEN** 模型调用成功且输出可解析
- **THEN** 接口 MUST 返回结构化菜品/食材结果，并标记处理状态为成功

### Requirement: 失败场景必须映射为统一错误码
系统 MUST 将上游模型错误映射为统一 AI 错误码并返回一致的错误响应结构，覆盖至少鉴权失败、限流、超时、上游服务异常与响应格式错误。

#### Scenario: 上游超时错误映射
- **WHEN** 模型调用超过服务端超时阈值
- **THEN** 接口 MUST 返回预定义的 AI 超时错误码与一致的错误响应体

### Requirement: 接口必须可观测且不泄露敏感信息
系统 MUST 记录视觉识别调用的请求追踪信息、耗时与结果状态用于监控；日志 MUST NOT 输出 API Key、用户原图明文敏感内容或完整上游认证头。

#### Scenario: 失败日志脱敏
- **WHEN** 模型调用失败并记录错误日志
- **THEN** 日志 MUST 包含请求追踪标识与失败类型，且 MUST NOT 包含敏感凭据
