# 首页 UI 架子（mobile-home-ui）

本规范定义 Flutter 客户端「首页」Tab 的结构与可验收行为（mock 数据、与主壳 Tab 的协作）。

---

## 规范要求

### Requirement: 首页 MUST 以可滚动纵向布局展示五大区块

已登录用户在选中「首页」Tab 时，主内容区 MUST 提供垂直滚动容器，并按自上而下顺序包含以下逻辑区块：（1）日期与标语问候区；（2）拍照主入口；（3）今日热量与进度卡片；（4）最近记录区块；（5）AI 建议区块。各区块之间 MUST 使用项目中间距 token 保持可读留白。

#### Scenario: 用户进入首页可见完整结构

- **WHEN** 用户已登录且选中「首页」Tab
- **THEN** 主内容区可纵向滚动，并依次展示上述五个区块，且不依赖后端网络请求即可渲染

### Requirement: 问候区 MUST 展示固定格式的日期与固定标语

问候区 MUST 展示一行日期文案，格式为中文惯用的「年、月、日」与「星期」组合（具体格式由实现选定，首版固定为单一展示样式即可）。问候区 MUST 在日期下方或相邻位置展示一句**前端写死**的标语文案（不要求可配置）。样式 MUST 复用 `AppTypography` 与 `AppColors` 等设计 token，避免在页面散落魔法数。

#### Scenario: 问候区展示日期与标语

- **WHEN** 用户查看首页顶部问候区
- **THEN** 可见日期行与固定标语行，且文字层级区分清晰（日期相对次要、标语相对突出）

### Requirement: 拍照主入口 MUST 可点击并预留扩展点

拍照主入口 MUST 使用现有 `CaptureCard`（或等价封装）呈现，且 MUST 响应点击（`onTap` 非空或可由父级注入）。首版 MAY 仅执行占位逻辑（如空实现或调试日志），MUST 保留后续接入相机页或路由的扩展方式（例如回调、命名路由占位），MUST NOT 在首版强制依赖相机权限或第三方插件。

#### Scenario: 用户点击拍照入口

- **WHEN** 用户点击拍照主入口
- **THEN** 应用不崩溃，且存在可替换的回调或路由挂点供后续接入真实流程

### Requirement: 今日热量卡片 MUST 使用 mock 数据并支持超额红色进度条

今日热量区块 MUST 展示「已摄入」「还可摄入」与「目标」等相关信息，数值首版 MUST 来自本地 mock（硬编码或临时模型均可）。进度条 MUST 表示已摄入相对日目标的比例；进度值 MUST clamp 在 0～100% 的展示范围内（即 `consumed / goal` 超过 1 时按满条显示）。当已摄入热量 **大于或等于** 日目标（`consumed >= goal`）时，进度条的前景/指示色 MUST 使用红色（danger 语义，来自 token）；当未达目标时，进度条指示色 MUST 使用常规主进度色（token）。该区块 MUST NOT 在首版调用后端 API。

#### Scenario: 未超目标时进度条为主色

- **WHEN** mock 数据中已摄入小于日目标
- **THEN** 进度条未满且指示色为常规主进度色（非红色）

#### Scenario: 达到或超过目标时进度条满条且为红色

- **WHEN** mock 数据中已摄入大于或等于日目标
- **THEN** 进度条显示为满条，且指示色为红色（danger）

### Requirement: 最近记录区块 MUST 含「查看全部」并仅展示一条记录

最近记录区块 MUST 具备区块标题「最近记录」与右侧「查看全部」操作。标题与操作 MUST 使用 `SectionTitle`（或等价组件）实现，且「查看全部」MUST 可点击。区块内 MUST 使用标准 `MealRecordCard`（或等价业务组件）展示**一条** mock 记录。点击「查看全部」MUST 将主壳当前 Tab 切换为「记录」Tab，MUST NOT 导航至记录 Tab 下的子路径或子路由栈。

#### Scenario: 展示单条最近记录

- **WHEN** 用户查看最近记录区块
- **THEN** 可见标题、「查看全部」以及一条餐食记录卡片

#### Scenario: 查看全部切换到记录 Tab

- **WHEN** 用户点击「查看全部」
- **THEN** 底部导航选中「记录」Tab，主内容区切换为记录 Tab 对应页面（可为占位页）

### Requirement: AI 建议区块 MUST 为独立业务展示且数据可 mock

AI 建议区块 MUST 使用 `AIInsightCard`（或基于 token 的等价封装）展示至少一段建议性文案，文案首版 MUST 为本地 mock。该区块 MUST NOT 调用后端或真实大模型接口。

#### Scenario: 用户可见 AI 建议卡

- **WHEN** 用户滚动至首页 AI 建议区块
- **THEN** 可见带有品牌风格的建议卡片及 mock 文案
