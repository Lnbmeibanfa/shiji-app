# 用户认证（短信登录）（user-auth）

本规范定义后端短信验证码登录、协议留痕、会话与退出相关能力。

---

## 规范要求

### Requirement: 系统必须提供短信验证码发送能力

系统 MUST 提供发送短信验证码接口，并在发送前执行发送频率控制。系统 MUST 在每次发送请求后记录 `sms_code_log`，包括手机号、验证码哈希、过期时间、状态与请求元信息。

#### Scenario: 发送验证码成功

- **WHEN** 用户提交合法手机号且未触发频控
- **THEN** 系统生成验证码并调用短信网关发送，同时写入一条 `sms_code_log` 记录，状态为待验证且过期时间为当前时间后 1 分钟

#### Scenario: 发送过于频繁被拒绝

- **WHEN** 同一手机号在 1 分钟窗口内重复请求发送
- **THEN** 系统 MUST 拒绝本次发送并返回频控错误，不得写入新的待验证验证码记录

### Requirement: 系统必须提供验证码登录并确保一次性消费

系统 MUST 提供手机号+验证码登录接口。登录校验时系统 MUST 仅使用该手机号最近一条有效且待验证的验证码记录，并校验过期状态、错误次数上限与验证码哈希。验证码错误最大次数 MUST 为 10 次。验证成功后该验证码记录 MUST 立即标记为已验证并不可再次用于登录。

#### Scenario: 验证码登录成功并消费验证码

- **WHEN** 用户提交的手机号与验证码匹配最近一条有效待验证记录，且验证码未过期且错误次数未超限
- **THEN** 系统 MUST 将该 `sms_code_log` 记录更新为已验证，并继续执行登录会话创建流程

#### Scenario: 验证码错误次数超限

- **WHEN** 同一条待验证验证码记录累计错误次数达到 10 次后再次校验
- **THEN** 系统 MUST 拒绝登录并将该验证码记录标记为不可继续校验状态

### Requirement: 系统必须在未同意协议时禁止登录

验证码登录前系统 MUST 校验协议同意状态。若用户未同意协议，系统 MUST 拒绝登录。系统 MUST 记录协议同意留痕至 `user_agreement_accept_log`，并支持在首次登录前仅使用手机号留痕。

#### Scenario: 未同意协议禁止登录

- **WHEN** 用户发起验证码登录但未提供协议同意
- **THEN** 系统 MUST 直接拒绝登录，不得创建用户会话

#### Scenario: 首次登录前记录协议同意

- **WHEN** 用户在验证码登录流程中提交协议同意且用户账号尚未建立
- **THEN** 系统 MUST 在 `user_agreement_accept_log` 记录包含手机号与协议版本的同意记录

### Requirement: 系统必须提供单设备会话与会话恢复能力

登录成功后系统 MUST 创建 `user_session` 作为会话权威记录，并签发 token 给客户端。会话有效期 MUST 为 30 天。系统 MUST 执行单设备策略：新会话建立时同用户其他有效会话全部失效。系统 MUST 提供会话恢复能力，仅当 token 存在、状态有效且未过期时恢复登录态。

#### Scenario: 单设备登录覆盖旧会话

- **WHEN** 用户验证码登录成功并创建新会话
- **THEN** 系统 MUST 使该用户历史有效会话失效，仅保留当前新会话为有效状态

#### Scenario: token 有效恢复会话

- **WHEN** 客户端携带有效 token 请求恢复会话
- **THEN** 系统 MUST 返回当前用户登录态信息

#### Scenario: token 失效或过期恢复失败

- **WHEN** 客户端携带不存在、已失效或已过期 token
- **THEN** 系统 MUST 拒绝恢复会话并返回未登录状态

### Requirement: 系统必须支持全部退出登录

系统 MUST 提供退出登录接口，并在执行退出时将当前用户所有有效会话更新为失效状态并记录退出时间。

#### Scenario: 用户执行全部退出

- **WHEN** 已登录用户调用退出接口
- **THEN** 系统 MUST 将该用户所有有效 `user_session` 记录状态置为失效并写入 `logout_at`
