# 服务端架构规范（server-architecture）

本规范定义食迹后端服务架构规则。

---

## 需求：服务端负责业务编排

所有业务逻辑必须在后端实现。

### 场景：AI饮食分析

- 当用户提交饮食记录
- 后端负责数据校验
- 后端调用 AI 服务
- 后端返回分析结果

---

## 需求：服务端采用分层架构

后端必须采用标准分层结构。

### 场景：新增接口

- Controller 负责接收请求
- Service 负责业务逻辑
- Repository 负责数据库访问
- 外部服务调用必须封装在独立模块

---

## 需求：接口返回统一结构

所有 API 返回统一格式。

### 场景：接口返回数据

返回结构必须为：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```
