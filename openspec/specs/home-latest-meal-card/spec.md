# 首页最近一餐卡片（home-latest-meal-card）

本规范定义首页最近一餐区块与只读接口的联动、展示与降级行为。

---

## 规范要求

### Requirement: 首页最近一餐卡片 MUST 在进入页面时请求最近一餐接口

客户端在用户进入首页后 MUST 主动调用最近一餐只读接口（`GET /api/meal-records/latest` 或等价封装）获取最新数据。请求状态 SHOULD 仅影响最近一餐区块，不得阻塞首页其它区块渲染。

#### Scenario: 进入首页即发起请求
- **WHEN** 用户进入首页 Tab
- **THEN** 最近一餐区块触发一次最近一餐接口请求

#### Scenario: 接口请求中首页仍可渲染
- **WHEN** 最近一餐接口尚未返回
- **THEN** 首页其余区块保持可见并可交互，最近一餐区块显示加载态或占位态

### Requirement: 最近一餐存在时 MUST 展示餐别、时间与整数热量

当接口返回 `data` 非空时，最近一餐卡片 MUST 同时展示餐别与记录时间，并展示热量值。热量展示 MUST 使用整数 `kcal` 文案，不保留小数位。

#### Scenario: 有效记录展示完整信息
- **WHEN** 接口返回一条最近一餐记录且包含热量
- **THEN** 卡片可见餐别、时间与整数热量文案（如 `520 kcal`）

### Requirement: 无记录时 MUST 展示引导并可跳转记录页面

当接口返回 `code=0` 且 `data=null` 时，最近一餐区块 MUST 展示“去记录一餐”的引导内容，且 MUST 提供可点击入口跳转到记录饮食页面。

#### Scenario: 无记录引导跳转
- **WHEN** 接口返回 `data=null` 且用户点击引导入口
- **THEN** 应用导航至记录饮食页面（如 `RoutePaths.recordMeal`）

### Requirement: 接口失败时 MUST 降级展示稳定占位

当最近一餐接口请求失败（网络异常、超时、服务端异常）时，最近一餐区块 MUST 展示稳定占位或降级文案，MUST NOT 导致首页崩溃或白屏。

#### Scenario: 请求失败降级
- **WHEN** 最近一餐接口调用失败
- **THEN** 最近一餐区块展示降级内容且首页其他区域正常显示
